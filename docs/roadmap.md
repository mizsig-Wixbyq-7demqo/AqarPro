# خارطة الطريق

## المنجز

- المهام 1–3: مخطط البيانات، RLS، بيانات التجربة، محرك المؤشرات، ولقطات المؤشرات (Snapshots).
- المهمة 4: إنشاء الحساب والدخول والخروج، تأكيد البريد، إعداد المنشأة، حماية المسارات، المنشأة النشطة، وصلاحيات الأدوار.
- المهمة 5: لوحة المحفظة الحية، بطاقات العقارات، صفحة العقار، إدارة الوحدات، العقود، وإنشاء جداول الدفعات تلقائيًا.
- المهمة 6: الدفعات الكاملة والجزئية، المصروفات وتصنيفاتها، مقارنة السوق، وسجل المؤشرات القابل للمراجعة.
- المهمة 7: مراجعة الأمان، وتشديد TypeScript، وتحسين تجربة العربية والجوال وإمكانية الوصول، وتشغيل الفحص والاختبارات والبناء النهائي.

## الجاهزية التشغيلية

- اكتمل نطاق المرحلة الأولى برمجيًا.
- يتطلب التشغيل المستضاف ضبط متغيرات Supabase و`NEXT_PUBLIC_SITE_URL` ثم تطبيق Migrations على مشروع Supabase المقصود.
- يبقى تحديث الاعتماديات ومراجعة سجل التدقيق عمل صيانة مستمرًا، وليس مرحلة بناء جديدة.

هذه الصفحة تسجل الميزات المؤجلة فقط، ولا تعد التزامًا بتنفيذها في المرحلة الأولى.

## المرحلة الثانية وما بعدها

- تكامل رسمي مباشر مع منصة إيجار بعد وجود صلاحية موثقة.
- EjarProvider كمحول مستقل بعد ثبوت واجهة رسمية.
- Web Scraping بعد مراجعة قانونية وتقنية صريحة، وليس كبديل افتراضي.
- القروض وDSCR والتدفق بعد التمويل.
- الصيانة وأوامر العمل.
- رفع المستندات وإدارتها.
- إشعارات واتساب والرسائل النصية والبريد الخارجي.
- تكامل البنوك والمحاسبة.
- ميزات ذكاء اصطناعي مساعدة؛ لا تستخدم لحساب المؤشرات الأساسية.
- Data Confidence Score متقدم.
- السيناريوهات المالية وSensitivity Analysis.
- تطبيق جوال مستقل.

DSCR اختصار **Debt Service Coverage Ratio — نسبة تغطية خدمة الدين**. Sensitivity Analysis يعني **تحليل الحساسية** لمعرفة أثر تغير الافتراضات على النتائج. كلاهما خارج المرحلة الأولى.
