# وثيقة متطلبات المنتج — AqarPro

## 1. الرؤية

AqarPro نظام عربي للمستثمر أو شركة إدارة الأملاك يجمع التشغيل والتحصيل والمصروفات ومقارنة السوق في سجل واحد قابل للمراجعة. القيمة الأساسية هي تحويل بيانات العقار اليومية إلى «نبض العقار» دون ادعاء دقة غير موجودة.

## 2. مستخدمو المرحلة الأولى

- مالك محفظة عقارية صغيرة أو متوسطة.
- مدير عقارات يدير العقارات والوحدات والعقود والعمليات.
- محاسب يسجل الدفعات والمصروفات ويراجع المؤشرات.
- مشاهد يحتاج الاطلاع دون تعديل.

## 3. أهداف المرحلة الأولى

يستطيع المستخدم:

1. إنشاء حساب ومنشأة معزولة عن بقية المنشآت.
2. إدارة العقارات والوحدات والمستأجرين والعقود.
3. إنشاء جدول الاستحقاقات وتسجيل دفعات كاملة أو جزئية.
4. تسجيل المصروفات المدفوعة أو المعلقة.
5. معرفة المستحق والمحصل والمتأخر وصافي التدفق.
6. إدخال بيانات مقارنة السوق يدويًا أو من CSV.
7. مشاهدة درجة الصحة التشغيلية، حالة التدفق النقدي، وفجوة الإيجار.
8. مشاهدة فرصة تقديرية لزيادة الدخل وسجل المؤشرات السابق.

CSV اختصار **Comma-Separated Values — ملف قيم مفصولة بفواصل**، ويستخدم للاستيراد المنظم دون تكامل مباشر مع مصدر السوق.

## 4. نبض العقار

يعرض ثلاثة مؤشرات رئيسية فقط:

- **Operating Health Score — درجة الصحة التشغيلية** من 100.
- **Cash Flow Status — حالة التدفق النقدي**: أخضر أو أصفر أو أحمر أو بيانات غير كافية.
- **Rent Gap Percentage — نسبة فجوة الإيجار عن السوق**، ولا تظهر عند غياب مقارنة صالحة.

تفاصيل الصيغ والحدود في [scoring-engine.md](scoring-engine.md).

## 5. نطاق بيانات السوق

- البداية التجريبية: الرياض؛ أحياء طويق وظهرة لبن والعارض.
- الأنماط التجريبية: غرفة وصالة، غرفتان وصالة، ثلاث غرف وصالة، أربع غرف وصالة.
- التصميم لا يقيد المدن أو الأحياء أو أنواع الوحدات بهذه القائمة.
- مصدر السوق يحتفظ بنوعه: عقد منفذ، إعلان طلب، أو تقدير يدوي.
- لا تخلط أنواع المصادر المختلفة داخل متوسط واحد.
- لا تخمّن الحقول المفقودة؛ المقارنة غير الصالحة تعيد: «لا توجد بيانات سوق كافية للمقارنة».

## 6. قصص الاستخدام الأساسية

### المنشأة والمحفظة

- بصفتي مالكًا، أُنشئ منشأة وأصبح مالكها الأساسي تلقائيًا.
- بصفتي عضوًا، لا أرى إلا بيانات المنشآت التي أنتمي إليها.
- بصفتي مالكًا، أدير الأعضاء والأدوار دون السماح لمدير بإزالة المالك الأساسي.

### العقود والتحصيل

- بصفتي مديرًا، أضيف عقدًا وأولّد استحقاقاته وفق دورية الدفع.
- بصفتي محاسبًا، أسجل مبلغًا كاملًا أو جزئيًا دون تعديل هوية المنشأة.
- بصفتي مستخدمًا، أرى حالة الاستحقاق: معلق أو جزئي أو مدفوع أو متأخر أو ملغى.

### السوق والمؤشرات

- بصفتي مديرًا، أدخل Benchmark يدويًا مع المصدر والفترة وعدد العينات.
- بصفتي مستخدمًا، أرى مقارنة الوحدة المؤجرة فقط إذا وجدت مقارنة مناسبة.
- بصفتي مستخدمًا، أستطيع مراجعة مدخلات ومخرجات كل Snapshot وإصدار الصيغة.

Benchmark يعني **قيمة مرجعية للسوق** تستخدم للمقارنة وليست ضمانًا لسعر يمكن تحصيله. Snapshot يعني **لقطة مؤشر محفوظة** تحتفظ بمدخلات الحساب ومخرجاته في وقت محدد.

## 7. متطلبات غير وظيفية

- RTL اختصار **Right-to-Left — اتجاه من اليمين إلى اليسار** على الجوال والكمبيوتر.
- عزل المنشآت يطبق داخل PostgreSQL عبر RLS وليس في الواجهة فقط.
- الأموال `numeric(15,2)`؛ لا تستخدم أنواع الأرقام التقريبية.
- جميع الصيغ حتمية وقابلة للاختبار والمراجعة ولا تستخدم الذكاء الاصطناعي.
- كل توقيت مخزن كـ`timestamptz` ويعرض وفق `Asia/Riyadh`.
- العمليات الحساسة المهمة تسجل في `audit_logs`.

## 8. معايير نجاح النسخة

- يمكن تشغيل المشروع من قاعدة فارغة وفق README.
- تنجح أوامر lint وtypecheck وtest وbuild.
- تفشل محاولات القراءة أو الكتابة العابرة للمنشآت.
- لا يملك viewer أي كتابة، ولا يتجاوز accountant أو manager حدود الدور.
- لا تظهر Rent Gap عند نقص بيانات السوق.
- تحتفظ كل لقطة بـ`formula_version = v1.0.0` ومدخلاتها ومخرجاتها.
