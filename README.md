# AqarPro — عقار برو

AqarPro تطبيق ويب عربي متعدد المنشآت لإدارة العقارات المؤجرة وقياس أدائها المالي والتشغيلي. تركز المرحلة الأولى على العقارات والوحدات والمستأجرين والعقود والدفعات والمصروفات، إضافة إلى «نبض العقار» بثلاثة مؤشرات: درجة الصحة التشغيلية، حالة التدفق النقدي، ونسبة فجوة الإيجار عن السوق.

الحالة الحالية: **اكتملت المهام السبع للمرحلة الأولى**. قاعدة البيانات وبيانات التجربة ومحرك «نبض العقار» والمصادقة وصلاحيات الأدوار ولوحة المحفظة وإدارة العقارات والوحدات والعقود والدفعات والمصروفات ومقارنة السوق وسجل المؤشرات جاهزة، واجتازت مراجعات الأمان وTypeScript والتصميم العربي والبناء النهائي.

## التقنية

- Next.js باستخدام App Router.
- TypeScript لضبط أنواع البيانات.
- Supabase للمصادقة وقاعدة البيانات.
- PostgreSQL لحفظ البيانات.
- Tailwind CSS للتصميم.
- واجهة عربية من اليمين إلى اليسار، والمنطقة الزمنية التجارية `Asia/Riyadh`.

RLS اختصار **Row Level Security — أمان مستوى الصفوف**، وهو طبقة قاعدة البيانات التي تعزل بيانات كل منشأة وتطبق الدور على كل عملية.

## متطلبات التشغيل

- Node.js إصدار `22.13.0` أو أحدث.
- npm إصدار حديث.
- Docker Desktop أو محرك Docker متوافق لتشغيل Supabase محليًا.
- منافذ Supabase الافتراضية `54321` إلى `54324` متاحة.

## الإعداد

```bash
npm install
cp .env.example .env.local
```

املأ في `.env.local` قيم مشروع Supabase المحلي أو المستضاف. لا تضع أي مفتاح حقيقي داخل المستودع. متغير `SUPABASE_SERVICE_ROLE_KEY` للخادم فقط، ولا يجوز استخدامه داخل المتصفح أو تسميته ببادئة `NEXT_PUBLIC_`.

أضف أيضًا `NEXT_PUBLIC_SITE_URL` بعنوان التطبيق حتى تعود رسالة تأكيد البريد إلى المسار الصحيح. لا يستخدم التطبيق مفتاح `service_role` في تسجيل الدخول أو إجراءات المستخدم.

في بيئة الإنتاج، يجب أن تكون `NEXT_PUBLIC_SITE_URL` قيمة HTTPS موثوقة؛ لا يعتمد رابط تأكيد الحساب على ترويسة `Origin` الواردة من المتصفح.

## التشغيل محليًا

```bash
npm run supabase:start
npm run db:reset
npm run dev
```

التطبيق يعمل افتراضيًا على `http://localhost:3000`، وSupabase Studio على العنوان الذي يعرضه أمر التشغيل المحلي.

لإيقاف الخدمات:

```bash
npm run supabase:stop
```

## قاعدة البيانات

تشغيل Migrations غير المطبقة:

```bash
npm run db:migrate
```

إعادة بناء قاعدة محلية فارغة ثم تشغيل جميع Migrations وملف Seed:

```bash
npm run db:reset
```

تشغيل Seed Data يتم حاليًا من خلال الأمر التالي:

```bash
npm run db:seed
```

ملف `supabase/seed.sql` يحتوي بيانات خيالية ثابتة لمدينة الرياض: منشأة بأدوارها الأربعة، عقاران، 23 وحدة، 19 عقدًا، 60 استحقاقًا، وبيانات مالية وسوقية ومؤشرات. لا تعدّل Migration سبق تنفيذها، بل أضف Migration جديدة باسم زمني أعلى.

فحص SQL عبر Supabase المحلي:

```bash
npm run db:lint
```

## الاختبارات والتحقق

```bash
npm run lint
npm run typecheck
npm test
npm run build
npm run audit:prod
```

اختبارات Migrations وSeed وRLS واختبارات تدفقات الوحدات والعقود والدفعات والمصروفات ومراجع السوق تستخدم PostgreSQL مضمّنًا مؤقتًا ولا تحتاج Docker:

```bash
npm run test:migrations
```

## بنية المشروع

```text
app/                     الدخول ولوحة المحفظة وصفحات التشغيل والماليات والتحليل
components/              عناصر الهوية والتنقل والبطاقات ومساحات العمل والنماذج
lib/auth/                الجلسة والمنشأة النشطة ومصفوفة قدرات الأدوار
lib/data/                استعلامات المحفظة والعقار والعمليات محددة الحقول
lib/domain/              تواريخ الرياض وتنسيق القيم وإنشاء جداول الدفعات
lib/supabase/            عملاء Supabase للمتصفح والخادم وتجديد الجلسة
lib/scoring/             محرك المؤشرات وحفظ Snapshots
docs/                    متطلبات المنتج والمعمارية والبيانات والأمان
supabase/config.toml     إعداد Supabase المحلي
supabase/migrations/     Migrations مرتبة زمنيًا وغير قابلة لإعادة الكتابة
supabase/seed.sql        بيانات تجربة عربية خيالية لمدينة الرياض
tests/                   اختبارات البنية والصلاحيات والمؤشرات وتدفقات التشغيل
```
